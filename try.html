PHASE 1: REQUIREMENT ANALYSIS

Step 1: System Features

  • Accept user input (two numbers and an operator)
  • Support addition (+), subtraction (−), multiplication (×), and division (÷)
  • Validate operator and input format
  • Prevent division by zero
  • Display calculation result
  • Reject invalid input

Step 2: Identify Users and System Needs

Users:

  • Students, beginners, or anyone needing quick arithmetic
  • Anyone using a terminal or command-line interface

System Purpose:

  • Perform arithmetic calculations and handle errors for invalid input

Step 3: Functional and Non-Functional Requirements

Functional Requirements:

  • Accept two numeric inputs and one operator from the user
  • Perform addition, subtraction, multiplication, and division
  • Prevent division by zero
  • Display the calculation result
  • Reject invalid inputs or unsupported operators

Non-Functional Requirements:

  • Run on a command-line interface
  • Respond to user input immediately (fast execution)
  • Handle errors gracefully without crashing
  • Simple and easy-to-use interface
  • Portable across systems with a C compiler

PHASE 2: SYSTEM ANALYSIS / DESIGN

Task: Identify requirements and design how the program will work

Design Steps:

  1. Input two numbers and an operator
  2. Validate input and operator
  3. Perform calculation based on the operator
  4. Display result

Examples:

Input: 5 + 3
Output: 8.00

Input: 5 / 0
Output: Error: division by zero

Flowchart

flowchart TD A[Start] --> B[Input two numbers and an operator] B --> C[Validate input and operator] C --> D{Is input valid?} D -- No --> E[Display error message] D -- Yes --> F[Perform calculation] F --> G[Display result] G --> H[End] E --> H

Algorithm in Pseudocode

START
   PROMPT "Enter expression (e.g. 5 + 3):"
   READ num1, op, num2

   IF input is not in the format number operator number THEN
       DISPLAY "Invalid input"
       END
   ENDIF

   IF op NOT IN ['+', '-', '*', '/'] THEN
       DISPLAY "Unsupported operator"
       END
   ENDIF

   IF op = '/' AND num2 = 0 THEN
       DISPLAY "Error: division by zero"
       END
   ENDIF

   IF op = '+' THEN
       result = num1 + num2
   ELSE IF op = '-' THEN
       result = num1 - num2
   ELSE IF op = '*' THEN
       result = num1 * num2
   ELSE IF op = '/' THEN
       result = num1 / num2
   ENDIF

   DISPLAY "Result: ", result
END

Algorithm: Digital Calculator

  1. Start
  2. Prompt the user: "Enter expression (e.g. 5 + 3)"
  3. Read first number → num1
  4. Read operator → op
  5. Read second number → num2
  6. Validate input: If input format is wrong → display "Invalid input" → End
  7. Validate operator: If op is not +, -, *, / → display "Unsupported operator" → End
  8. Check division by zero: If op = '/' and num2 = 0 → display "Error: division by zero" → End
  9. Perform calculation:
    • If op = '+' → result = num1 + num2
    • If op = '-' → result = num1 - num2
    • If op = '*' → result = num1 * num2
    • If op = '/' → result = num1 / num2
  10. Display "Result: " followed by result
  11. End
